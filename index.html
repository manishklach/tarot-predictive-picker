Deterministic Gather for Live Inference State

Systems and methods for deterministic gather of hierarchically managed key-value state for autoregressive neural network inference.

This invention treats KV state as logically managed shared state rather than a flat tensor, and introduces a deterministic gather path that converts irregular, tiered, shared, and format-diverse live inference state into execution-ready tiles, streams, or descriptors for downstream compute.

Core framing

Preparing logically managed KV state for execution

The architecture is not framed as larger local memory, generic caching, compression in isolation, or paged KV allocation alone. Residency-first hierarchical management supplies the control substrate. Deterministic gather is the operative mechanism.

  • Logical KV objects, not flat per-sequence tensors
  • Hierarchical metadata for placement, ownership, and representation
  • Shared prefixes and branch-aware state lineage
  • Mixed-tier retrieval with execution-ready output generation

Problem

Inference state becomes the systems problem

Long-context autoregressive inference places increasing pressure on key-value state management. Historical KV grows with context length, session count, branch count, and resumability requirements. In practical serving systems, the dominant bottleneck is often not arithmetic alone, but preparation of historical state for arithmetic.

Conventional approaches duplicate shared prefixes, rely on incidental cache behavior, or store history in flat per-sequence tensors that are difficult to share, relocate, compress, migrate, or reconstitute predictably. As state fragments across tiers and formats, execution preparation becomes irregular.

Long-context pressure

KV footprint grows faster than preferred hot-tier capacity.

Redundant retrieval

Shared prompt history is repeatedly materialized or recomputed.

Generic caching limits

Implicit locality mechanisms do not understand sequence lineage or branch value.

Tier fragmentation

Historical state spreads across local, package, host, and remote memory.

Execution unpredictability

Irregular state must still be made regular before attention can consume it.

Shared-state control gap

Prefix sharing, branch divergence, and resumable sessions are poorly represented by flat storage.

Core Invention

Deterministic gather as the central mechanism

Logical KV objects

KV state is treated as logically managed live inference state. Objects are named independently of current physical placement and may be shared across sessions or branches.

Residency-first hierarchy

A hierarchical metadata plane tracks ownership, placement, representation, version state, and tier residency. This plane is the substrate that makes deterministic gather correct and policy-governed.

Execution-ready output

A deterministic gather engine resolves logically distributed state and emits execution-ready tiles, buffers, stream descriptors, DMA chains, or front-end command structures for downstream attention compute.

Architecture Overview

Control-plane resolution, data-plane preparation

The architecture separates control-plane resolution from compute consumption. Hierarchical metadata resolves logical identity, ownership, tier placement, representation, and version state. Deterministic gather then compiles a concrete execution path that regularizes irregular live state into a form that kernels can consume without metadata traversal.

Hierarchical Metadata Manager

Maintains logical identity, source descriptors, epochs, and residency records.

Prefix / Branch Ownership Manager

Represents shared lineage, divergence boundaries, and branch-local suffix ownership.

Residency Controller

Places objects across hot, warm, cold, and optional remote tiers under explicit policy.

Compression / Representation Manager

Tracks representation class, transform path, and tier-appropriate storage form.

Deterministic Gather Engine

Compiles gather plans, binds sources, schedules overlap, and emits execution-ready output.

Compute Interface

Supplies normalized tiles or descriptor-form artifacts to attention compute.

Why It Matters

Architectural significance in deployed inference systems

Long-context serving

Historical state can remain logically available without forcing uniform hot-tier residency.

Multi-session inference

Shared prompt regions can be referenced as common logical objects rather than duplicated state.

Shared prompt reuse

Prefix sharing becomes a first-class state-management operation, not an incidental optimization.

Resumable sessions

Logical identity persists even when state migrates across tiers or representations.

QoS-aware placement

Policy can prioritize latency-sensitive sessions and gather-critical regions explicitly.

Distributed support

Remote ownership and local replication decisions can be integrated into the same gather path.

Method / Execution Flow

From logical request to execution-ready output

01

Logical state request

An inference step identifies the historical KV state required by the downstream attention operator.

02

Metadata resolution

Logical identity, ownership lineage, tier placement, representation class, and version state are resolved through the control plane.

03

Gather-plan compilation

Source descriptors are bound, transform paths are selected, and segment ordering is compiled into a deterministic gather plan.

04

Mixed-tier retrieval

Hot-tier reads, warm-tier transfers, cold-tier fetches, optional remote sources, and transforms are overlapped under plan control.

05

Execution-ready generation

The gather path emits a regular artifact: materialized tile, staged buffer, stream descriptor, DMA chain, or command structure.

06

Compute consumption

The compute unit receives normalized input without direct traversal of the metadata hierarchy.

Deterministic Gather for Live Inference State

The purpose of the gather path is not only retrieval. It is regularization: converting logically shared, fragmented, compressed, and tier-distributed live inference state into a stable execution contract for the compute plane.
